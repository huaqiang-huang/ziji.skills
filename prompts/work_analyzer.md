# Work 分析 Prompt

## 任务

你将收到用户关于“自己”的原材料。

你的任务是只从材料中提炼 **可执行的工作方式**，生成 Work 层输入。

不要做人格模仿，不要堆形容词，不要输出空泛优点。

---

## 核心提取维度

### 1. 任务拆解方式

- 用户接到任务后通常先做什么
- 习惯先澄清还是先试做
- 怎么拆阶段
- 怎么判断“可以开始做了”

### 2. 交付规范

- 输出偏好：文档、代码、清单、结论顺序
- 对“完成”的判断标准
- 是否强调验证、复盘、风险说明

### 3. 协作习惯

- 怎么和别人对齐
- 怎么给反馈
- 怎么处理不明确需求
- 怎么处理别人改动或并行协作

### 4. 领域知识与工作边界

- 常做的任务类型
- 擅长和负责的领域
- 明显不想接或不擅长的工作

### 5. 工具与资料来源

优先识别：

- 代码库
- 技术方案
- 设计文档
- 复盘
- 笔记
- 任务清单
- PR 评论

---

## 权重规则

- 高权重：主动写的长文、代码、设计方案、复盘、长期笔记
- 中权重：PR 评论、会议纪要、聊天里较长的任务讨论
- 低权重：口头一句带过、零碎标签、临时抱怨

---

## 输出要求

输出必须尽量落到：

- 明确步骤
- 明确标准
- 明确偏好
- 明确边界

禁止输出：

- “注重质量”
- “认真负责”
- “逻辑清晰”

这类空泛表述，除非有非常具体的行为支撑。
