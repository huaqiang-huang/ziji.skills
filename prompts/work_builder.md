# Work 生成模板

## 任务

根据 Work 分析结果生成 `work.md`。

Work 层要回答的问题是：

- 这个人通常怎么推进任务
- 这个人怎样定义高质量交付
- 这个人喜欢怎样协作
- 这个人在哪些领域有经验

---

## 模板

```markdown
# {name} — Work

## 1. 任务推进方式

### 接到任务时
{步骤}

### 开始执行前
{会先确认什么}

### 交付时
{输出要求}

## 2. 输出与质量标准

### 默认输出形式
{文档/代码/清单/说明的偏好}

### 完成标准
{如何判断已经完成}

### 常见自检项
{常用检查点}

## 3. 协作习惯

### 对齐方式
{如何沟通需求和边界}

### 反馈方式
{如何指出问题}

### 处理并行协作
{如何避免互相踩}

## 4. 领域与边界

### 擅长领域
{列表}

### 常处理的任务
{列表}

### 不优先接的事
{边界}

## 5. 经验知识库

{尽量具体的规则、经验、踩坑结论}

## 6. Work 使用说明

- 写代码时遵循这里的方式
- 写文档时遵循这里的结构
- 处理任务时先按这里的步骤推进
- 遇到边界外的问题，按边界说明收缩范围
```
