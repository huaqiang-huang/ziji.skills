# 四层增量 Merge Prompt

## 任务

你将收到：

1. 现有的 `work.md`
2. 现有的 `persona.md`
3. 现有的 `principles.md`
4. 现有的 `recovery.md`
5. 新的原材料

你的任务是判断新内容应该更新到哪一层，并输出增量 patch。

原则：

- 只追加或修正，不整段重写
- 有冲突时必须显式指出
- `best` 模式下，低质量旧习惯不能覆盖主规则
- 新资料只落到相关层，不要四层一起重写

---

## 归类规则

| 信息类型 | 归入 |
|---------|------|
| 任务拆解、交付方法、协作方式、领域知识 | `work.md` |
| 表达方式、互动姿态、压力表现、口头习惯 | `persona.md` |
| 稳定原则、优先级、边界、取舍规则 | `principles.md` |
| 换路策略、自检清单、失败恢复、冲突处理 | `recovery.md` |

---

## 冲突规则

发现冲突时输出：

```text
⚠️ 发现冲突
- 现有内容：{...}
- 新内容：{...}
- 建议处理：保留 / 覆盖 / 作为旧习惯附注
```

---

## 输出格式

```text
=== work.md 更新 ===
{patch 或 无更新}

=== persona.md 更新 ===
{patch 或 无更新}

=== principles.md 更新 ===
{patch 或 无更新}

=== recovery.md 更新 ===
{patch 或 无更新}
```
